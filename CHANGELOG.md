# Changelog

All notable changes to **polyalign** are documented here.
This project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html) and uses pre-release tags (e.g. `0.1.0a1`) per [PEP 440](https://peps.python.org/pep-0440/).

## [0.1.0a2] - 2026-05-29 (pre-alpha, post-audit honest-marketing patch)

### Honest-marketing corrections (post-/compact 4-agent audit findings)

- **Cycle-consistency**: documented that at the default `cycle_threshold=0.0` the predicate is structurally True for any non-negative transport plan; a non-trivial threshold will be the v0.2 default. Vertex/CLAIM docstrings + new test `test_polytope_3_bundles_cycle_threshold_discriminates` make this falsifiable.
- **Star projection**: for `N >= 3`, `extract_polytope_vertices` is a star projection from bundle 0 (not a full pairwise-consistent clique). Docstring + CLAIM `[non-CLAIM]` row added; full clique enumeration deferred to v0.2.
- **OTCP column rename**: README ablation table's `otcp_coverage` column renamed to `otcp_q` (it is the split-conformal threshold, not empirical coverage); table footnote added.
- **top5 ceiling footnote**: README clarifies that `top5 = 0.208` in the planted regime is the bound `5 / 24` (100% precision over emitted vertices), not 20.8% accuracy on the full target set.
- **Bussmann claim softening**: `[CLAIM]` for Matryoshka reproduction tightened to "non-increasing reconstruction error in prefix length, consistent with Bussmann monotonicity" (the synthetic test uses identity-target reconstruction).
- **OTCP coverage claim softening**: `[CLAIM]` tightened from "marginal coverage >= 1 - alpha" to match the actual test condition (within 15% slack on >= 50% of synthetic trials).
- **gavagai score demoted**: `[non-CLAIM]` row added — the gavagai pairwise indeterminacy score (`gavagai.score.gavagai_score`) is NOT invoked in v0.1.0a2; only the `HookedSAEBundle` type adapter is wired. Score integration deferred to v0.1.1.

### Code hardening

- Sinkhorn-OT plan now raises `RuntimeError` if `ot.sinkhorn` returns a plan containing NaN/Inf (defensive against user-supplied extreme `reg`).
- `extract_polytope_vertices` / `alignment_polytope` no longer duplicate the marginal-quantile arithmetic; both now call the single `_marginal_q_from_plans` helper. Dead `calib_pairs` and `pooled_plan` allocations removed.
- `MAMBA_SSM_HOOK_SITE: Final[str] = "out_proj_out"` constant exported from `polyalign._types`, anchoring the recurrentlens convention as a code-level value rather than docstring-only.
- Ablation `ece` metric on the `planted` regime now uses `correct = (edge matches planted permutation)` (measures calibration shape against known truth); `random` regime keeps `correct = ones` (Case C, vacuously correct, documented).

### Reproducibility

- `scripts/run_ablation.py` now honors `POLYALIGN_REPRO_DATE` env var for the `env_stamp.date` field, so the JSON can be regenerated byte-equal across days when the env is set.
- README ablation table re-rendered from the new `results/v0.1.0a2_ablation.json`.

### CHANGELOG correction

- The v0.1.0a1 entry incorrectly described the conditional OTCP API as `otcp_calibrate(..., conditional=True)`. The actual signature is the separate function `polyalign.conformal.otcp.otcp_calibrate_conditional(plans, bundles, calib_pairs, *, alpha=...)`; this CHANGELOG and CHANGELOG-referenced docs are updated.

### Tests

- 79 tests prior; v0.1.0a2 adds `test_polytope_3_bundles_cycle_threshold_discriminates` (cycle predicate discriminates at threshold > 0). Total >= 80.

### Provenance / NOTICE

- NOTICE split into two sections: "Code used (license obligations apply)" — gavagai, recurrentlens, foldconsensus, POT — and "Design patterns referenced (no code vendored)" — hybridlens, saelet, foldgauge, mosaicraft-active-vision, weightlock. The latter group remains attributed for honest provenance but is no longer presented as licensed-derivative work.

## [0.1.0a1] - 2026-05-29 (pre-alpha)

### Initial release

- `polyalign.SAEBundle` type: `model_id` + `architecture` (transformer/ssm/hybrid) + `layer` + decoder/encoder.
- `polyalign.alignment.sinkhorn.sinkhorn_align(bundle_a, bundle_b, reg, cost)`: Sinkhorn-OT pairwise alignment (POT backend).
- `polyalign.alignment.pairwise.pairwise_alignments(bundles)`: N x (N-1) / 2 pairwise alignments.
- `polyalign.matryoshka.MatryoshkaWrapper`: prefix-nested alignment over `[d/8, d/4, d/2, d]` 4 stages.
- `polyalign.conformal.otcp.otcp_calibrate(P, calib_pairs, alpha)`: split conformal coverage q (marginal mode default).
- `polyalign.calibration.pava`: vendored PAVA monotone isotonic regression from foldconsensus (Apache-2.0).
- `polyalign.calibration.ece`: equal-frequency-bin Expected Calibration Error.
- `polyalign.polytope.vertices.extract_polytope_vertices(...)`: novel polyalign core — Pareto-front top-k vertices with cycle-consistency.
- `polyalign-lint` CLI (typer).
- CI matrix: Python 3.10 / 3.11 / 3.12 on Linux.
- Dependabot security-updates only (version-updates limit 0, per `feedback_github_org_hygiene_2026-05-25`).
- Honest-marketing CI grep: ban list defined in `.github/banned-phrases.txt` rejects unverifiable comparative claims; the list itself is not duplicated in user-facing docs (avoids the self-reference trap in `feedback_ci-grep-bre-vs-ere-2026-05-24`).
- Tests >= 60, coverage >= 85%.

### Known limitations

- All `v0.1.0a1` ablation metrics are computed against **synthetic ground truth** (Case C activated at S0 due to live cross-model SAE inference being infeasible inside the build session). All synthetic feature pairs carry the `[DEMO]` prefix in `datasets/ground_truth_v0.1.0a1.jsonl`.
- Live cross-model SAE extraction (Llama-3 + Gemma-3 + Mamba-2) is **deferred to v0.1.1**.
- Poincare ball / hyperbolic cost matrix, OKLab visualization, and Muon optimizer are **deferred to v0.2**.
- Conditional-coverage OTCP per architecture-pair is implemented but the marginal mode is the default in this release. Switch via the separate function `polyalign.conformal.otcp.otcp_calibrate_conditional(plans, bundles, calib_pairs, *, alpha=...)`. (Documented incorrectly as a keyword arg in the original v0.1.0a1 changelog; corrected in v0.1.0a2.)

### Provenance / what polyalign composes

Code used (license obligations apply):

- `gavagai` (MIT) — `HookedSAEBundle` type adapter only (`src/polyalign/backends/gavagai_bridge.py`). gavagai's pairwise indeterminacy score is NOT invoked in v0.1.0a1 / v0.1.0a2; score integration is deferred to v0.1.1.
- `recurrentlens` (Apache-2.0) — SSM hook convention `out_proj_out` (recorded as `MAMBA_SSM_HOOK_SITE` constant in v0.1.0a2; live extraction deferred to v0.1.1).
- `foldconsensus` (Apache-2.0) — vendored PAVA monotone isotonic regression in `polyalign/calibration/pava.py`.
- POT (MIT) — Sinkhorn-Knopp solver (`ot.sinkhorn`).

Design patterns referenced (no code vendored):

- `hybridlens`, `saelet`, `foldgauge`, `mosaicraft-active-vision`, `weightlock` (all Apache-2.0).
